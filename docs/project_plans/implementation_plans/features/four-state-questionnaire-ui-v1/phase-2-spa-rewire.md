# Phase 2: SPA Read/Write Rewire

**Column conventions**: `Estimate` is story points, never Effort. `Model`: `sonnet` (primary Claude —
this phase is **kept on primary**, not delegated to ica, per decisions block §7: "highest-risk seam").
`Effort`: `adaptive` | `extended`. `Provider`: `claude` for every implementer task in this phase; `codex`
for the cross-family review row.

**Why this phase stays on primary, not ica.** Per the decisions block's routing rationale (§7) and
risk hotspots R1/R2 (§5), **now joined by R8/R9 from the cross-family review** (F-1/F-4,
`.claude/findings/four-state-questionnaire-ui-cross-family-review.md`): this phase touches the
`RadioNodeList` guard boundary (R1), rewrites safety-adjacent mutual-exclusion logic (R2) that
force-writes across every `immediateSafetyNames` field, rewires the workflow/depth/safety-count
consumers that a checkbox-only conversion would silently break (R8/F-1 — the review's confirmed
BLOCKER, missed by the same-family review pass), and fixes a safety-review auto-assertion whose
premise this feature invalidates (R9/F-4). A regression here is not a markup typo — it is a functional
read/write defect in code paths that gate an immediate-safety-screen checkbox and the safety banner
itself. This is the plan's genuine risk concentration — **9.5 of 20 total points, re-estimated up
from 5 of 14** after the cross-family review found the F-1/F-3/F-4 gaps — and is deliberately not
delegated.

**Ordering constraint (binding)**: this phase requires `src/facts/fieldState.js` (P1, including P1-03's
`isPresent`/`isAssessed` predicates) to exist and be tested **before** any of the rewrites below begin —
every rewritten function delegates its state-mapping decision to that module (FR-8, FR-16) rather than
inlining logic, mirroring how `setSimpleField:1466` already delegates to `toTri()` today.

**Evidence base for every anchor below**: `src/app.js` as read at plan-authoring time (worktree HEAD);
cross-checked against `.claude/worknotes/four-state-questionnaire-ui/spike-legC-surface.md §3` and
`decisions-block.md §5` (R1/R2). Line numbers are exact as of this plan's authoring commit and should
be re-confirmed at execution time if intervening commits touch `src/app.js`.
